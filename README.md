# mk-portal-notes

本仓库用于归档和发布多个独立 HTML 页面。这些页面内容互不关联，各自承担不同的展示或功能用途，不针对任何特定域名或网站。

## 项目简介

这是一个静态页面集合仓库，主要存放以 HTML 格式编写的独立页面。每个页面可视为一个独立的“入口笔记”，用于呈现信息、工具或说明性内容。仓库本身不包含后端逻辑或动态数据，适合通过 GitHub Pages 或类似服务直接托管。

## 目录说明

仓库根目录下包含以下内容：

- `index.html` — 可选的总览页面，用于索引或导航到其他页面（如存在）
- `*.html` — 多个独立的 HTML 页面文件，每个文件对应一个独立内容
- `assets/` — 存放页面所需的公共资源，如 CSS、JavaScript 或图片文件（如存在）
- `docs/` — 可选文档或说明文件

各页面之间无依赖关系，可独立访问。

## 页面归档说明

- 每个 HTML 页面均为自包含文件，包含完整的结构、样式和脚本（或引用公共资源）。
- 页面内容不绑定任何具体域名或外部服务，可离线查看或迁移至任意静态托管环境。
- 文件命名建议使用英文小写字母、数字和连字符，便于 URL 访问。
- 若页面需要外部资源（如字体、图标库），建议使用通用 CDN 或下载至本地 `assets` 目录。

## 维护说明

- 新增页面时，直接在根目录创建新 `.html` 文件，或按功能放入子目录。
- 更新现有页面时，请保持文件命名不变，以免影响已有引用或链接。
- 不建议在页面中嵌入跟踪脚本、广告或需要特定域名才能访问的内容，以确保页面长期可用。
- 欢迎提交 Issue 或 Pull Request 提出改进建议或报告问题。
